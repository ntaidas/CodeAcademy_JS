Asinchroninis programavimas

Asinchroninio programavimo idėja yra ta, kad kol vykdosi vienas kodas, galima leisti vykdytis ir kitam kodui. Kitaip pasakius, atlikinėti keletą veiksmų vienu metu.

  1. setTimeout()
  2. async ir await
  3. try catch finally
  4. Promise (resolve, reject)
  5. then catch finally

Async ir await

Rašant kodą asinchroniškai kartais prireikia luktelti atsakymų iš prieštai buvusių funkcijų. Tam, kad sulauktume atsakymo, reikia uždelsti kodą. Tai galima padaryti su setTimeout(callback, time);
Persistengiant su setTimeout'ais ar kitais callback'ais sukuriamas vadinimasis callback Hell, to pasekoje nukenčia kodo skaitomumas, suprantamumas ir tęstinumas/tobulinimas.
Tam, kad išvengtume vadinamojo callback Hell, mes galime kurti asinchronines funkcijas. Jos kuriamos prirašant raktinį žodį "async".
Norint nurodyti, kad reikia kažkurio veiksmo atsakymo asinchroninėje funkcijoje sulaukti, galime prieš tą veiksmą prirašyti raktinį žodį "await", kuris nurodo, kad reikia laukto atsakymo iš to veiksmo, prieš vykdant tolimesnį kodą.

Syntax

async function name(){ veiksmai1... await veiksmai2... veiksmai3 priklausantys nuo veiksmai2... }
let name = async () => { veiksmai1... await veiksmai2... veiksmai3 priklausantys nuo veiksmai2... }

Try Catch Finally

Try, Catch ir Finally naudojami norint aprašyti error'ų valdymą gražia, patogia ir efektyvia sintakse.
Try bloke vykdomas kodas. Jeigu Try bloke įvyksta klaida - vykdomas Catch blokas. Galiausiai vykdomas Finally blokas.
Net jeigu Try bloke įvyksta klaida, kodas Try bloke einantis už to error'o, visviena vykdysis.

Syntax

try{ code } catch{ errorInCode } finally{ doWhatever }

Promise ir then catch finally

Rašant asinchroninį kodą, naudojami Promises.
Promise'as pažada grąžinti kažkokį atsakymą, kurio Promiso iškvietimo momentu mes negauname.
Promise mums arba grąžins atsakymą, kurį yra pažadėjęs (resolve'ins) arba dėl kažkokios priežasties norimo atsakymo negrąžins, grąžins užklausos atmetimo informaciją (reject).
Kadangi Promise mums atsakymo negrąžina iš karto, mes su tuo laukiamu atsakymu veiksmus galime atlikti tik tuomet, kada jis yra grąžinamas (then). Jeigu grąžinamas yra error'as (reject) arba then bloke įvyksta klaida, ją galime pagauti (catch) ir kažką su juo atlikti. O galų gale vykdyti kažką betkokiu atveju (finally).

Syntax

Promise((resolve, reject) => { if(good){ resolve('answer') } else { reject('error') } })
