API-Application Programing Interface

Uzklausos. Isiuncia jusu uzklausa is aplikacijos(front-end) i serveri(back-end).Ir jei viskas tvarkoje grazina duomenis pagal jusu uzklausa. Jei kazkas negerai, grazina atitinkama error zinute.

Fetch

Komanda/funkcija/metodas, kurios pagalba galime siusti uzklausas i serveri.

Fetch veikimo eiga

  1. Fetchinti(siusti uzklausa)duomenis kazkuriuo metodu
    • GET - gauti informacija
    • POST - pateikti informacija
    • DELETE - trinti informacija
    • PATCH - redaguoti informacija
    • PUT - redaguoti informacija
  2. Apdoroti gauta informacija(vykdosi kai nera error )
  3. Naudoti gauta informacija(vykdosi kai nera error )
  4. Pagaunam ir kazka atliekam su error(vykdosi tik error atveju)
  5. Galiausiai (finally) dar kazka atliekam( vykdosi betkokiu atveju)
