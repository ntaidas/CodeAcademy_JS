Moduliai

Moduliai yra atskiri kodo fragmentai iskaidyti skirtinguose failuose.

Tuose moduliuose gali buti ivairus kodas. Nuo duomenu iki funkciju, klasiu, metodu ir ect...

JS klases turi buti atskiri moduliai. Kitaip pasakius - vienam faile neturetu buti aprasytos kelios klases.
HTMLe prigungtas scriptas turi tureti nurodyta modulio tipa.
HTML failas, turi buti paleistas kaip serveris, nes kitu atveju neleis vykdyti JS failo kodo.

Export ir Import

Failuose esanti informacija gali buti iskeliama (exportuojama) ir galima kazkokia informacija isikelti is kito failo(importuoti)

Export

Informacija galima eksportuoti dvejais budais.
1) export default - numatytasis/pagrindinis exportas gali buti tik vienas viename faile.
2) export - papildomas exportas/ai, kuriu gali buti daug viename faile.

Importas

Informacija gali importuoti ivairiais budais.
1) import sameNameAsExportDefault from ./someFile - gaunama Default export informacija
2) import differentNameAsExportDefault from ./someFile - gaunama Default export informacija
3) import { sameNameAsExport, sameNameAsExport2, ... } from ./someFile - gaunama nurodyto pavadinimo export informacija (ne export default, o paprasto export)
4) import { sameNameAsExport as norimasNaujasPavadinimas, sameNameAsExport2, ... } from ./someFile - gaunama nurodyto pavadinimo export informacija, kuria siame faile vadinsi kitu sugalvotu pavadinimu.
5) import pagrindinisImport,{ nePgrImport, nePgrImport2 as kitasPavadinimas, ...} from ./someFile - galima vienu importu importuoti ir export default ir tiesiog export.
